### 7. 回答引用由「页码」改为「文件名 + 行号」

**涉及文件：** `src/retrieval.py`、`src/prompts.py`、`src/questions_processing.py`、`src/api_requests.py`

**修改内容：**

- **检索结果**：`retrieve_by_company_name` 与 `retrieve_all` 的每条结果增加 `file_name`（来自 metainfo）、`lines`（来自 chunk 的 `[start, end]`）；无 `content.pages` 时 `retrieve_all` 按 `content.chunks` 遍历并带 `file_name`/`lines`。
- **RAG 上下文格式**：`_format_retrieval_results` 由「Text retrieved from page {page}」改为「Text retrieved from file "{file_name}", lines {start}-{end}」，无 `lines` 时退回使用 page。
- **Prompt 与 Schema**：在 `prompts.py` 中新增 `SourceRef`（file_name, line_start, line_end）；所有答题 Schema 的 `relevant_pages: List[int]` 改为 `relevant_sources: List[SourceRef]`，说明与示例统一为「文件名 + 行号范围」。
- **校验与引用**：`_validate_page_references` 改为 `_validate_source_references`，校验 LLM 返回的 (file_name, line_start, line_end) 是否出现在检索结果中，不足则补足、过多则截断；`_extract_references` 仅接收校验后的来源列表，返回 `[{"file_name", "line_start", "line_end"}, ...]`，不再使用 pdf_sha1/page_index。
- **答案与提交**：`get_answer_for_company` 使用 `relevant_sources` 与 `references`（文件名+行号）；`answer_details` 存 `relevant_sources`；`_post_process_submission_answers` 的 references 改为 `{"file_name", "line_start", "line_end"}`；`process_comparative_question` 去重键改为 `(file_name, line_start, line_end)`；`api_requests.get_answer_from_rag_context` 兜底结构改为 `relevant_sources`。

**目的：** 回答与提交中的引用以「文件名 + 行号」呈现，便于精确定位到原文行范围，与当前分块结构（chunk 含 lines）一致。
